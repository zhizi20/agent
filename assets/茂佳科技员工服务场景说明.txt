二、常见员工服务场景（训练营归纳）
1. **入职与培训**
   - 新员工入职培训、岗位技能训练
   - 产品知识（TV/Monitor/商显/ODM）普及
   - 数字化工具指引（Coze、内部系统等模拟场景）

2. **行政后勤**
   - 食堂、宿舍、厂区通勤
   - 晚班/夜班交通接驳
   - 多基地出差后勤

3. **办公与 IT**
   - 会议室预约与使用
   - 工位网络、VPN、在线会议
   - 办公设备报修

4. **制造一线（模拟场景）**
   - 产线排班信息获取
   - 劳保用品、休息区
   - 跨班组沟通

5. **研发办公（模拟场景）**
   - 测试实验室预约
   - 跨时区与海外基地协作
   - 技术资料与培训资源入口

6. **职业发展**
   - 产品知识、外语、数字化技能类培训需求
   - 内部讲师/分享会

三、反馈分类与典型责任映射（模拟）
| 类别 | 典型问题 | 建议对接 |
|------|----------|----------|
| 行政后勤 | 食堂排队、接驳车 | 行政 |
| 办公环境 | 会议室占用、工位环境 | 行政 / 设施管理 |
| IT 支持 | 网络不稳、系统登录 | IT |
| 培训发展 | 课件入口、产品知识 | HR / 培训负责人 |
| 流程工具 | 预约系统不好用 | IT / 流程 Owner |
| 跨基地协作 | 时差会议、信息同步 | 各部门主管 / IT |

四、紧急程度判断参考
- **高**：安全、大面积停工、数据泄露风险
- **中**：影响效率但有临时方案（如网络不稳但可热点）
- **低**：体验优化、长期建议

五、茂佳特色反馈主题（模拟，供 Bot 识别）
- 「想了解 TCL 和茂佳的关系」→ 培训/文化类，可链到产品知识培训
- 「想懂 ODM 和 TV/商显区别」→ 培训发展类
- 「惠州/成都/海外基地协作不便」→ 流程/工具或管理沟通类
- 「产线夜班回宿舍交通」→ 行政后勤类
- 「希望参观智能制造车间」→ 培训/文化类（模拟）

六、员工心声 Bot 输出原则
1. 先分类，再判紧急程度，再给责任建议
2. 同类反馈合并归纳高频问题
3. 回复话术礼貌、具体、不含虚假承诺（如「下周一定改好」）
4. 涉及个人隐私、人事评价、劳动纠纷 → 提示升级 HR 人工处理

七、禁止事项
- 不输出真实姓名、工号、电话
- 不对员工个人贴标签
- 不处理未脱敏的敏感投诉
- 不代替 HR 做正式人事答复

【边界】本文为训练营模拟场景，不代表真实员工政策；具体制度以公司正式文件为准。
