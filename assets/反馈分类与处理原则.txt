【模拟资料 · 反馈处理原则】

一、反馈分类建议
| 类别 | 示例 |
|------|------|
| 行政后勤 | 食堂、宿舍、通勤 |
| 办公环境 | 会议室、网络、工位 |
| 培训发展 | 课件、产品知识、技能训练 |
| 流程工具 | 系统不好用、指引不清 |
| 其他 | 无法归类的建议 |

二、紧急程度参考（模拟）
- 高：涉及安全、大面积影响工作
- 中：影响效率但有临时替代方案
- 低：体验优化类、长期改进类

三、建议责任部门（模拟映射）
- 食堂/通勤 → 行政
- 培训课件/产品培训 → HR / 培训负责人
- 会议室 → 行政 / IT
- 网络 → IT

四、智能体输出结构
1. 反馈分类表
2. 高频问题摘要
3. 紧急程度判断（说明依据）
4. 建议责任部门
5. 处理建议（可执行步骤）
6. 员工回复话术（礼貌、不含虚假承诺）

五、严格禁止
- 不得输出真实员工姓名、工号、电话
- 不得对员工个人做评价或标签化
- 不得处理涉及个人隐私的敏感投诉细节
- 不得承诺具体整改日期（除非用户明确提供）
